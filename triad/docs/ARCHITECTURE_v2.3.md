## 变更日志（v2.2 → v2.3）

v2.3 是**架构精简版本**。核心变更：

1. **移除 ComfyUI 多模态执行层**：图像/视频生成不再嵌入核心，未来通过 MCP 插件扩展
2. **移除 Claude Code Daemon**：代码执行由 Hermes 内置工具集替代，不再依赖外部闭源 CLI
3. **llama.cpp 改为检测模式**：安装脚本不再自动编译 CUDA，改为检测用户自行安装的 llama-server
4. **核心聚焦**：OpenClaw Gateway（通信）+ Hermes Orchestrator（认知编排）+ llama.cpp（本地推理）

---

## 一、核心洞察：为什么 v2.3 是必要的

v2.2 的架构假设是"全链路多模态内容生成工作站"，包含图像/视频/TTS/代码执行。但实践中发现：

- **ComfyUI 是完整的专业工作流引擎**，有自己的节点生态，塞进 Triad 后变成"寄生式"耦合
- **Claude Code 是闭源产品**，接口随时会变，和 Triad "数据不出站"的理念矛盾
- **llama.cpp 编译** 对用户的 GCC/CUDA 版本极其敏感，成为安装的最大卡点

**v2.3 的核心决策**：Triad 只做一件事——**认知编排**。通信（OpenClaw）、路由（Hermes）、推理（llama.cpp），其他一切通过 MCP/HTTP 接入。

## 三、核心子系统详解

### 3.1 OpenClaw Gateway（TypeScript / Node.js）

职责：**入口、通信、UI、系统监控**

| 模块 | 功能 | 端口 |
|------|------|------|
| WebSocket Server | 浏览器多标签页实时通信 | 18080 |
| REST API | 模型路由、健康检查 | 18080 |
| 系统探针 | GPU 显存/温度/利用率监控 | 18080 |
| 启动面板 | 一键启停所有服务 | WebUI |

---

### 3.2 Hermes Orchestrator（Python / asyncio）

职责：**意图理解、策略选择、模型路由、执行调度、技能进化**

| 模块 | 功能 |
|------|------|
| 动态 5 步流水线 | 角色注入 → 策略路由 → 模型分发 → 执行反馈 → 反思固化 |
| SwarmExecutor | 蜂群并发：研究员 + 写手 + 审校并行 |
| VRAMScheduler | 显存跷跷板：-ngl 99 ↔ 0，GPU/CPU 动态切换 |
| ModelRouter | 云端 API 和本地 llama.cpp 的统一路由 |
| SkillCrystallizer | 高分任务自动固化为 Markdown + YAML 配方 |

---

### 3.3 llama.cpp（C++ / CUDA）

职责：**本地 LLM 推理引擎**

| 特性 | 说明 |
|------|------|
| GGUF 格式 | 统一量化模型格式 |
| CUDA 加速 | -ngl 999 全层 GPU 卸载 |
| OpenAI API 兼容 | /v1/chat/completions 标准接口 |
| 多模态 | 支持 Qwen2-VL、LLaVA 视觉模型 |

**v2.3 变更**：安装脚本不再自动编译 llama.cpp。用户自行安装后，脚本检测运行状态。

## 五、安装流程（精简为 10 步）

```bash
# 1. 克隆仓库
git clone https://github.com/iprisoner/Triad.git
cd Triad/triad

# 2. 一键安装（检测 llama-server，不自动编译）
sudo ./triad_manager.sh install
```

安装步骤：
1. 系统环境检查（WSL2 / Ubuntu 22.04 / GPU）
2. 配置国内镜像源（apt / pip / npm / docker）
3. 安装 Docker CE + nvidia-container-toolkit
4. Python 3.10 + Hermes (mind/) 依赖
5. Node.js 18 + OpenClaw (host/openclaw) + WebUI (webui) 依赖
6. **检测 llama-server**（用户自行安装，脚本不编译）
7. Qdrant 向量数据库（Docker）
8. .env 环境文件
9. Web UI 构建
10. 安装后测试验证

## 六、使用模式

### 单体模式（轻骑兵）

```
用户输入 → 角色注入 → 模型路由 → llama-server 本地推理 → 输出
```

### 蜂群模式（特种部队）

```
用户输入 → 蜂群调度 → 研究员∥写手∥审校并发 → 结果聚合 → 输出
```
